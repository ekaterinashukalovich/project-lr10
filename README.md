# Разработка ML-сервиса для предсказания одобрения ипотеки

## Цель работы

Освоить полный цикл разработки прикладного ML-сервиса: от обработки данных и обучения модели машинного обучения до создания API, пользовательского интерфейса, логирования, метрик и упаковки приложения для запуска.

## Описание задания

Необходимо разработать консультирующий сервис, который предсказывает вероятность одобрения или отказа в выдаче ипотеки на основе данных клиента.
Сервис реализован в виде веб-приложения на базе FastAPI. API принимает данные клиента, выполняет предобработку признаков, передает их в обученную ML-модель и возвращает результат предсказания: статус заявки и вероятность одобрения.

## Реализованная функциональность

В проекте реализованы следующие основные компоненты:

1. **ML-часть**

   * выполнена очистка данных от аномалий;
   * удалены некорректные значения, включая записи с нереалистичным возрастом клиентов;
   * применено масштабирование числовых признаков с помощью StandardScaler;
   * выполнено кодирование категориальных признаков с помощью OneHotEncoder;
   * выполнен feature engineering: логарифмирование признаков дохода и суммы кредита;
   * проведен отбор признаков: удален признак person_age из-за высокой корреляции с опытом работы;
   * обучены модели LogisticRegression, RandomForestClassifier и GradientBoostingClassifier;
   * лучшей моделью выбран RandomForestClassifier;
   * качество модели оценивалось с помощью метрик ROC-AUC, Precision и Recall;
   * итоговая модель сохранена в формате .pkl вместе с этапами предобработки.

2. **Backend**

   * backend реализован на FastAPI;

   * создана структура приложения с файлами [main.py](http://main.py), [settings.py](http://settings.py) и модулями в папке services;

   * в [main.py](http://main.py) создано FastAPI-приложение и настроен CORS;

   * добавлены основные эндпоинты:

     * POST /upload-model/ — загрузка ML-модели в формате .pkl;
     * POST /predict/ — предсказание по JSON-данным;
     * POST /predict-from-csv/ — пакетное предсказание по CSV-файлу;

   * в [settings.py](http://settings.py) реализовано хранение загруженной модели в памяти приложения;

   * в model_service.py реализованы загрузка модели и выполнение предсказаний;

   * в [preprocess.py](http://preprocess.py) реализована предобработка данных: удаление дубликатов, лишних колонок, пропусков и приведение признаков к нужным типам;

   * в csv_service.py реализована обработка CSV-файлов и добавление результатов предсказания в таблицу;

   * при наличии колонки loan_status в CSV дополнительно вычисляется метрика ROC-AUC;

   * в [validators.py](http://validators.py) реализована проверка загружаемых файлов модели и CSV-файлов;

   * результат предсказания возвращается пользователю в виде статуса approved или rejected.

3. **Frontend**

   * в проекте присутствует папка static, содержащая файлы пользовательского интерфейса:

     * index.html;
     * app.js;
     * style.css.

   * интерфейс используется для взаимодействия пользователя с backend-сервисом.

4. **Инфраструктура**

   * проект содержит Dockerfile и docker-compose.yml;
   * зависимости описаны в pyproject.toml и requirements.txt;
   * используются FastAPI, Uvicorn, Pandas, NumPy, Scikit-learn, Joblib и Pytest.

## Описание работы сервиса

Пользователь вводит данные клиента: возраст, пол, образование, доход, опыт работы, тип владения жильем, сумму кредита, цель кредита, процентную ставку, долю кредита от дохода, длину кредитной истории, кредитный рейтинг и наличие предыдущих дефолтов.

После отправки данных сервис выполняет следующие шаги:

1. Валидирует входные данные.

2. Преобразует данные в DataFrame.

3. Выполняет предобработку признаков.

4. Проверяет наличие всех необходимых колонок.

5. Передает данные в ML-модель.

6. Получает предсказание класса:

   * 1 — заявка одобрена;
   * 0 — заявка отклонена.

7. Получает вероятность одобрения.

8. Возвращает пользователю результат в JSON-формате.

## Оценка качества модели

Для оценки качества моделей использовались метрики классификации. Основной метрикой была выбрана ROC-AUC, так как она хорошо подходит для оценки бинарной классификации при несбалансированных классах.

По результатам экспериментов лучшей оказалась модель Random Forest:

* ROC-AUC: 0.97;
* Precision: 88%;
* Recall: 76%.

Это показывает, что модель достаточно хорошо различает заявки, которые могут быть одобрены или отклонены.
