# project-lr10
Разработка ML-сервиса для предсказания одобрения ипотеки

## ML Engineer part 
1. Предобработка данных (Data Preprocessing)
- Очистка данных от аномалий и привод их к единому формату.
- Удаление аномалий убрали людей кому 144 лет
- Стандартизация (Scaling) В ColumnTransformer использовали StandardScaler(). Это превращает числа так, чтобы они имели среднее 0 и разброс 1 (важно для линейных моделей и метрик).
- Кодирование (Encoding): В ColumnTransformer использовали OneHotEncoder(). Это превращает текст ("RENT", "OWN") в цифры (0 и 1).
2. Feature engineering и отбор признаков
- Были создады новые полезные признаки и удаление лишних.
Были взяты "кривые" распределения дохода и суммы кредита и сглажены логарифмом. Это помогло моделям (особенно линейной регрессии) работать лучше.
- Feature Selection (Отбор признаков).
Мы посмотрели на корреляцию (0.95 между возрастом и опытом) и удалили person_age, чтобы убрать дублирование информации.
3. Выбор и обучение моделей
Были Обучены 3 разных алгоритма из которых был выбран лучший.
- Выбор моделей: Использовали LogisticRegression, RandomForestClassifier, GradientBoostingClassifier.
- Мы обучили их в Pipeline, чтобы данные сразу проходили через очистку и скалирование.
Мы использовали class_weight='balanced', так как у нас мало одобренных кредитов (22%).
4. Оценка качества
Проверка насколько хорошо модели работают.
- Метрика ROC-AUC: Это главный показатель. Мы получили 0.97 для Random Forest.
- Classification Report: Посмотрели на Precision (точность) и Recall (полноту).
- Precision: 88% (если модель сказала "да", то в 88% случаев это правда).
- Recall: 76% (модель нашла 76% всех хороших клиентов).
5. Сериализация модели
Сохранили готовый продукт в файл, чтобы его можно было использовать в будущем.
Мы сохранили не просто модель, а весь Pipeline (включая настройки скалера и энкодера).

## Backend part
Backend реализован на FastAPI и отвечает за загрузку ML-модели, предобработку данных клиентов и выполнение предсказаний по одобрению ипотеки.

### Структура backend

```text
backend/
├── main.py
├── settings.py
└── services/
    ├── csv_service.py
    ├── model_service.py
    ├── preprocess.py
    └── validators.py
````

### Основные файлы

#### `main.py`
Главный файл backend-приложения. В нём создаётся объект FastAPI, настраивается CORS и описываются API endpoints.
Backend содержит три основных endpoint:
* POST /upload-model/ — загрузка ML-модели в формате .pkl;
* POST /predict/ — предсказание по JSON-данным;
* POST /predict-from-csv/ — предсказание по CSV-файлу.

#### `settings.py`
Файл с глобальными настройками приложения.
В нём создаётся объект settings, внутри которого хранится загруженная ML-модель.
После загрузки модели через /upload-model/ она сохраняется в памяти приложения и используется для дальнейших предсказаний.

#### `services/model_service.py`
Модуль для работы с ML-моделью.
Он содержит две основные функции:
* load_model_from_bytes() — загружает модель из бинарного файла;
* predict() — выполняет предсказание на подготовленных данных. Если модель ещё не была загружена, возвращает ошибку.

#### `services/preprocess.py`
Модуль предобработки данных.
В нём задаются признаки, которые используются моделью:
* числовые признаки;
* категориальные признаки;
* целевая переменная loan_status.

Функция preprocess() выполняет обработку входных данных:
* удаляет дубликаты;
* удаляет неиспользуемые колонки;
* приводит числовые признаки к числовому типу;
* удаляет строки с пропущенными значениями.

#### `services/csv_service.py`
Модуль для обработки CSV-файлов.
Он подготавливает данные из CSV к предсказанию, выбирает нужные признаки и добавляет результаты модели обратно в таблицу.
Если в CSV-файле есть колонка loan_status, дополнительно вычисляется метрика ROC-AUC.

#### `services/validators.py`
Модуль с функциями проверки загружаемых файлов.
Он проверяет:
* что файл модели является бинарным;
* что файл с данными является CSV-файлом.


Сначала пользователь загружает модель через endpoint /upload-model/. После этого можно отправлять данные клиентов через JSON или CSV. Backend предобрабатывает данные, передаёт их в модель и возвращает результат: approved или rejected.
